# 🏠 Smart Home IoT

Aplicativo mobile de automação residencial desenvolvido com **React Native + Expo**. Permite monitorar temperatura e umidade em tempo real, controlar iluminação remotamente e visualizar o histórico de leituras — tudo via protocolo **MQTT** sobre WebSocket e persistência em **Supabase**.

---

## 📱 Funcionalidades

- 🌡️ **Monitoramento em tempo real** de temperatura (°C) e umidade (%) via gauges circulares animados
- 💡 **Controle de iluminação** remoto (ligar/desligar) via MQTT
- 📈 **Dashboard com Gráficos** de histórico de leitura usando `victory-native`
- 📊 **Histórico em lista** salvo no banco de dados Supabase (últimas 20 entradas)
- 🔌 **Reconexão automática** ao broker MQTT com modal de status de erro
- 🌙 **Interface dark mode** com design moderno e responsivo
- 📱 **Navegação** integrada com React Navigation

## 🔌 Tópicos MQTT

| Tópico       | Direção      | Descrição                             |
|--------------|--------------|---------------------------------------|
| `casa/temp`  | Subscribe    | Temperatura atual em °C               |
| `casa/umid`  | Subscribe    | Umidade relativa do ar em %           |
| `casa/luz`   | Sub + Publish| Estado da luz (`"1"` = on, `"0"` = off)|

## ⚙️ Pré-requisitos

- [Node.js](https://nodejs.org/) 18+
- [Expo CLI](https://docs.expo.dev/get-started/installation/) (`npm install -g expo-cli`)
- Aplicativo **Expo Go** no celular (iOS/Android) ou emulador configurado
- Conta no [HiveMQ Cloud](https://www.hivemq.com/mqtt-cloud-broker/) (broker MQTT gratuito)
- Conta no [Supabase](https://supabase.com/) com tabela `sensor_data` criada

---

## 🗄️ Configuração do Supabase

Crie a tabela `sensor_data` no seu projeto Supabase com o seguinte schema:

```sql
create table sensor_data (
  id          bigint generated by default as identity primary key,
  topic       text        not null,
  value       text        not null,
  created_at  timestamptz not null default now()
);
```

## 🚀 Como Rodar

### 1. Clone o repositório

```bash
git clone https://github.com/seu-usuario/MyIoTProject.git
cd MyIoTProject
```

### 2. Instale as dependências

```bash
npm install
```

### 3. Configure as variáveis de ambiente

Copie o arquivo de exemplo e preencha com suas credenciais:

```bash
cp .env.example .env
```

Edite o `.env`:

```env
MQTT_HOST=xxxxxxxxxxxxxxx.s1.eu.hivemq.cloud
MQTT_PORT=8884
MQTT_PATH=/mqtt
MQTT_USER=seu_usuario_mqtt
MQTT_PASS=sua_senha_mqtt

SUPABASE_URL=https://xxxxxxxxxxx.supabase.co
SUPABASE_KEY=sua_anon_key_aqui
```

### 4. Inicie o servidor de desenvolvimento

```bash
npx expo start --clear
```

Escaneie o QR Code com o **Expo Go** no celular, ou pressione:
- `a` — abre no emulador Android
- `i` — abre no simulador iOS
- `w` — abre no navegador (web)

## 🧩 Componentes e Telas

### `HomeScreen` (Em `App.js`)
Tela inicial contendo o controle de iluminação, os gauges e a lista de histórico.

### `DashboardScreen` e `Dashboard`
Nova tela (acessível através do botão "Ver Dashboard" na Home) contendo gráficos de linha (`victory-native`) mostrando a evolução temporal da temperatura e umidade baseada nos últimos registros do banco de dados.

### `Gauges`
Exibe dois gauges circulares animados lado a lado para **temperatura** (vermelho) e **umidade** (azul), usando `react-native-circular-progress-indicator`.

### `LightControl`
Botão com ícone de lâmpada (`MaterialCommunityIcons`) que alterna entre os estados **ligada** (amarelo) e **desligada** (cinza). Ao tocar, publica `"1"` ou `"0"` no tópico `casa/luz`.

### `HistoryCard`
Card que exibe um registro histórico do banco, mostrando o tópico MQTT, o valor recebido e a data/hora formatada em `pt-BR`.

### `StatusModal`
Modal de erro exibido quando a conexão com o broker MQTT falha. Oferece as opções **"Tentar Novamente"** (reconecta imediatamente) e **"Tentar Mais Tarde"** (fecha o modal).

## 🔧 Serviços

### `MQTTService`
Encapsula a biblioteca Paho MQTT via `react_native_mqtt`. Expõe os métodos:
- `connect(config, onMessage, onConnect, onFailure)` — conecta ao broker com SSL
- `subscribe(topic)` — inscreve em um tópico
- `publish(topic, message)` — publica uma mensagem

### `supabaseService`
Funções utilitárias para interação com o Supabase:
- `saveReading(topic, value)` — insere um novo registro na tabela `sensor_data`
- `getReadings()` — busca as últimas 20 leituras em ordem decrescente de data

## 🐛 Problemas Conhecidos

- **`[Reanimated] createAnimatedPropAdapter is no longer necessary`** — aviso de deprecação do Reanimated 4, inofensivo. Será resolvido em versões futuras das dependências.
- **`Error: Got unexpected undefined`** no Metro — pode ocorrer durante hot-reload após mudanças de módulo. Reinicie com `npx expo start --clear` para limpar o cache.
